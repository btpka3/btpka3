⚠️ Shadow DOM 注入脚本的注意事项（展开查看）

问题现象：通过 fetch() 加载外部完整 HTML 并注入 Shadow DOM 后，被嵌入页面（如本文件）中的按钮点击事件不生效，脚本中的 document.getElementById() 找不到 Shadow DOM 内的元素。

原因分析：

  1. innerHTML 插入的 <script> 标签不会被浏览器执行，这是浏览器的安全设计——只有 HTML parser 解析时遇到的脚本才会执行。
  2. 即使通过 createElement('script') 手动创建并追加脚本，脚本中的 document 仍然指向主页面 document，无法访问 Shadow DOM 内的 DOM 节点。

解决方案（在宿主页面 shadow-dom-embed.html 中实现）：

  1. 样式注入：将 <style> 通过 appendChild 插入 Shadow DOM，CSS 正常生效。
  2. DOM 注入：将 <body> 内容通过 DocumentFragment 克隆后插入。
  3. 脚本执行：手动 createElement('script') 创建新脚本并追加到 Shadow DOM，浏览器会执行它。
  4. 作用域修复：在目标脚本之前注入 shim，劫持 document.getElementById / document.querySelector 等方法，让它们先查 Shadow DOM，再 fallback 到主 document。
