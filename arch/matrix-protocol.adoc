Matrix 是一个开放的、去中心化的通信协议，旨在解决即时通讯（IM）领域的“孤岛效应”问题。它允许不同服务器上的用户相互通信，就像电子邮件系统一样，任何人都可以搭建自己的服务器，同时还能与其他服务器上的用户自由交流。


1. 核心理念：打破孤岛。 允许不同的服务器之间互通消息
2. 技术架构与工作原理
** 全局聊天室状态图：
Matrix 将每个聊天室（Room）视为一个共享的状态图。所有的消息、成员变动、元数据修改都被记录为事件（Events）。
** 有向无环图（DAG）：
聊天记录不是简单的线性列表，而是一个有向无环图。每个新事件都引用前一个事件，确保历史记录的完整性和防篡改性。
** 最终一致性：
当用户在服务器 A 发送消息时，该消息会被复制到参与该聊天室的所有其他服务器（服务器 B、C...）。所有服务器最终会达成一致的状态。如果网络分区导致暂时不一致，协议有一套冲突解决机制来合并数据。
端到端加密（E2EE）：
Matrix 原生支持基于 Olm 和 Megolm 算法的端到端加密（类似 Signal 协议）。这意味着只有通信双方能解密消息，甚至连运行服务器的管理员也无法查看聊天内容。


== 关键组成部分


* Homeserver (家庭服务器)	用户注册、存储数据和路由消息的核心节点。最著名的开源实现是 Synapse (Python)，还有 Dendrite (Go, 更轻量) 和 Conduit (Rust)。
* Client (客户端)	用户用来聊天的软件。最官方且功能最全的是 Element (原名 Riot)，此外还有 FluffyChat, Cinny, Nheko 等多种选择。
* Bridge (桥接器)	连接 Matrix 与其他非 Matrix 网络（如微信、钉钉、Slack、Telegram）的中间件，实现跨平台消息同步。
* Identity Server	(可选) 用于查找用户的邮箱或手机号对应的 Matrix ID，帮助建立联系。
