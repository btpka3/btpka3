== AGENTS.md 是做什么的？
在 multi-agent（多智能体）系统或复杂单智能体系统中，AGENTS.md 通常包含以下核心内容：

* 角色定义 (Persona)：定义每个 Agent 是谁（例如：“你是一个资深 Python 工程师”或“你是一个挑剔的产品经理”）。
* 目标与职责 (Goals & Responsibilities)：明确该 Agent 负责解决什么问题，它的成功标准是什么。
* 协作流程 (Workflow)：描述多个 Agent 之间如何交互（例如：研究员先找资料 -> 交给作家写草稿 -> 交给编辑审核）。
* 上下文约束 (Context)：提供项目背景、特定领域的知识库摘要，帮助 Agent 理解环境。
* 工具绑定提示：虽然具体的工具代码不在这里，但这里会说明该 Agent 应该使用哪些 Skills。

简单来说：

* 如果 Skill 是 Agent 手里的“锤子、锯子”。
* 如果 Rule 是工地上贴的“必须戴安全帽、禁止吸烟”。
* 那么 AGENTS.md 就是告诉 Agent：“你是木匠，你的任务是用锤子和锯子造一把椅子，做完后交给油漆工，过程中要遵守安全规定。”
