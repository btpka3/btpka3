== 沙箱隔离


QoderWork 本身运行在你本机的 secure workspace 里，没有使用 Docker 或 VM 级别的沙箱。它的隔离策略主要依赖：

* 文件系统隔离：工作目录限定在 ~/.qoderwork/workspace/<session-id>/ 内，Agent 只能在这个目录里读写
* 安全策略：运行时注入的安全规则（比如你看到的禁止 rm -rf /、禁止硬编码密钥等），通过 prompt 层面约束行为
* 审计日志：所有操作被记录，供事后 review

本质上是 软隔离（通过策略和目录限制），而不是硬隔离（容器/虚拟机内核级隔离）。对于本地个人使用场景足够，但不适合多租户场景。
