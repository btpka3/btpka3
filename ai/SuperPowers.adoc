

* link:https://github.com/obra/superpowers[SuperPowers]


Superpowers 是一款由 Jesse Vincent(obra) 打造的开源 AI 编程工作流框架，核心理念是 ‌Process over Prompt（流程大于提示词）‌。
它不直接生成代码，本质上是一套为 Claude Code、OpenAI Codex CLI、Cursor、Gemini CLI 等编程 AI 设计的"插件式技能库", 为其装上一套“纪律与护栏”，
使其像资深工程师一样先思考、再规划、后编码、必验证，从而解决 AI 急于动手、误解需求、代码质量参差等问题.
工程化增强系统、强制落地软件工程规范、把AI当做严格约束的初级工程师。


核心工作流技能‌：该工具通过一组可组合的“技能”（Skills）来约束代理行为，主要包含以下关键环节 ：‌‌

* ‌头脑风暴（brainstorming）‌：在写代码之前，通过一次只问一个问题的方式，逐步澄清需求、探索方案、确定设计，并将设计文档保存到 docs/superpowers/specs/ 中。
* using-git-worktrees（使用 Git Worktree） — 设计获批后，创建独立的分支工作区，确保隔离开发
* ‌编写计划（writing-plans）‌：将设计拆解为可执行的小任务，每个任务包含精确文件路径和验证步骤。
* ‌测试驱动开发（test-driven-development）‌：强制执行 RED-GREEN-REFACTOR 循环，先写失败测试，再写最小代码，测试前写的代码会被删除 。‌‌
* requesting-code-review（代码审查与验证‌） — 任务之间进行评审，按严重级别报告问题，严重问题阻止进度。
* ‌子代理驱动开发（subagent-driven-development）‌：为每个任务启动新的子代理，进行两阶段审查（规格合规性、代码质量），支持自主工作数小时不偏离计划 。‌‌⭕️：是同一个会话内分派任务。
* finishing-a-development-branch（完成开发分支） — 所有任务完成后，验证测试、提供合并/PR/保留/丢弃的选项，清理 worktree。


仅覆盖 软件开发的： 需求 -> 编码 -> 测试 -> 交付 ，强制 TDD, 两阶段审查。



== claude 中安装
[source,shell]
----
/plugin install superpowers@claude-plugins-official

tree ~/.claude/plugins/cache/claude-plugins-official/superpowers/
----



== 它是如何生效的？
自动生效，不需要特殊步骤或指令。 关键机制如下：

. Plugin 注册 — .claude-plugin/plugin.json 定义了插件元数据，Claude Code 启动时自动发现并加载已安装的插件

. SessionStart Hook — 核心入口在 hooks/hooks.json，注册了 SessionStart 事件钩子。当 Claude Code 会话启动（startup/clear/compact）时，自动执行 hooks/session-start 脚本

. 上下文注入 — session-start 脚本读取 skills/using-superpowers/SKILL.md 的内容，将其以 JSON 格式注入到 Claude Code 的 additionalContext 中（通过 hookSpecificOutput 字段）。这段上下文中包含了极其重要的指令：

.. "You have superpowers"
.. 完整的 using-superpowers skill 内容
.. 要求 Agent 在响应任何内容之前检查是否有适用的 skill

. Skill 自动发现 — Claude Code 的 skills 系统会自动发现插件目录下的 skills/ 子目录中的 skill。每个 skill 是一个包含 SKILL.md 的目录，带有 YAML frontmatter（name、description）。description 字段决定了何时自动激活该 skill

. Skill 优先级 — 每个用户的消息到达时，Agent 必须根据 description 判断是否有 skill 适用（哪怕只有 1% 的可能性也要调用 Skill 工具）。这是强制性的，不是可选的

== 是否需要特殊指令激活？
不需要。 安装后自动生效。具体来说：

. 会话启动时，SessionStart hook 自动注入 using-superpowers 的上下文
. 用户输入任何消息时，Agent 根据 skill 的 description 自动判断是否需要调用相应 skill
. 例如用户说"加个登录功能"，brainstorming skill 的 description 会匹配，Agent 自动调用它
. 用户说"修复这个 bug"，systematic-debugging skill 会被触发
唯一的"手动"方式是通过 /plugin 命令管理插件本身，日常使用不需要任何特殊指令。
