将mcp 桥接成cli 命令行。


== 安装

[source,shell]
----
curl -fsSL https://mcp2cli.dev/install.sh | sh
mcp2cli --help
----

== 示例

=== 示例1：本地 arthas mcp 服务

先本地安装 arthas， 并attach 到一个java进程上

[source,shell]
----
# 本地安装 arthas, 且确保版本 >= 4.1.0
curl -L https://arthas.aliyun.com/install.sh | sh

# 启动arthas 并attach 到一个java进程上
./as.sh

# 检查arthas mcp 启动状况
curl -s -i -X POST \
  -H "Accept: application/json, text/event-stream" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"curl-demo","version":"1.0"}}}' \
  http://localhost:8563/mcp

----

更多curl示例参考： link:../java/arthas.adoc[arthas个人小结]

注意：如果想多个连续执行 多个 curl 命令 ，是需要后续 curl命令都要携带首次 http response header 的 mcp-session-id 的。

[source,shell]
----
###################################################################### 初始化
# 创建一个 mcp2cli 桥接mcp 的配置项
mcp2cli config init --name arthas \
    --endpoint http://localhost:8563/mcp

mcp2cli config list

# http连接复用：用 daemon 模式启动 arthas cli, 完成 mcp 连接的 initialize, 以便能保持会话，以及 请求头的 mcp-session-id 的携带。
mcp2cli daemon start arthas

###################################################################### 验证1： 通过 mcp2cli 命令
mcp2cli use arthas
mcp2cli arthas --help

# 列出 arthas mcp 可用工具： 方式一
mcp2cli arthas ls
# 列出 arthas mcp 可用工具： 方式二
mcp2cli arthas tool --help
mcp2cli arthas tool list

# 查看完整 arthas mcp 相关信息，包含tool 的 json schema ,以便确认各个 tools 的入参
mcp2cli arthas inspect --json | jq -C '.' | less -R
mcp2cli arthas inspect --json | jq '.data.inventory.tools[] | select(.id == "sysenv") | {id, description, inputSchema}'

# 调用命令
mcp2cli arthas tool call sysenv --help
# 查看所有环境变量
mcp2cli arthas tool call sysenv 2>/dev/null | grep '^{' | jq
# 查看指定环境变量:PATH
mcp2cli arthas tool call sysenv --arg envName=PATH 2>/dev/null | grep '^{' | jq
mcp2cli arthas tool call sysenv --args-json '{"envName":"PATH"}' 2>/dev/null | grep '^{' | jq

###################################################################### 验证1： 通过 mcp2cli 创建的 快捷命令（软连接）

# 创建 命令软连接  ~/.local/bin/arthas ，链接到 .local/bin/mcp2cli 。
mcp2cli link create --name arthas

# 后续 `arthas` 命令就相当于 前面示例的 `mcp2cli arthas` 命令了，
# 执行 `arthas` 命令时 mcp2cli 会根据命令行的可执行程序名（这里就是 "arthas") 来决定调用哪个 配置的 mcp 。
arthas ls
arthas tool call sysenv --arg envName=PATH 2>/dev/null | grep '^{' | jq
----
